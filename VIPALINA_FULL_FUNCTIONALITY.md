# ПОЛНЫЙ ФУНКЦИОНАЛ БОТА ВИПАЛИНА

## 1. ОБЩАЯ ИНФОРМАЦИЯ

**Vipalina** — комплексная система автоматизации работы VIP-отдела Zerocoder University.

**Задачи:** Автоматизация онбординга студентов, распределение между менеджерами, мониторинг SLA, генерация отчётов, управление чатами, синхронизация данных.

**Архитектура (двойной клиент):**
- **User Client** (@ultralina_zerocoder) — создание групп, личные сообщения
- **Bot Client** (@zerocoder_ultralina_bot) — inline-кнопки, команды

**Уровни доступа:**
- **VIP-менеджеры:** команды онбординга, отчёты по своим студентам, управление чатами
- **Руководитель:** полный доступ, сравнительная аналитика, настройка маппинга курсов
- **Дежурные:** приём студентов, базовые отчёты

## 2. АВТОМАТИЧЕСКИЙ ОНБОРДИНГ СТУДЕНТОВ

**Мониторинг:** Бот отслеживает чат VIP-отдела (-1001755644531), реагирует на сообщения от @zerocoder_vipgetcourse_bot, извлекает данные студента (имя, GetCourse ID, Telegram, email, телефон, курс).

**Распределение менеджеров:**
- **VIP очередь:** 6 менеджеров, round-robin
- **Luxury очередь:** 7 менеджеров, отдельный счётчик
- Индексы сохраняются в "Queue_State", восстанавливаются после перезапуска

**Уведомление менеджера:** Полные данные студента + кнопки [Принять] (запуск онбординга) / [Пропустить] (передача следующему)

**Этапы онбординга:**
1. **NocoDB:** поиск студента, извлечение GetCourse ID, менеджера
2. **KPI Ultra:** данные из "Общий список new" (имя, курс, статус, тариф)
3. **Трекер:** создание Google Sheet, листы курса, формулы, dropdown, доступ студенту
4. **Чат:** создание группы, добавление участников, invite-ссылка, приветствие
5. **Випалина:** запись всех данных (имя, ID, менеджер, курс, трекер, invite, финансовые даты)

**Ошибки приватности:** Создание чата "Менеджер+Бот", инструкции, уведомление в VIP-чат, автоповтор после исправления.

**Трекинг:** Лист "Onboarding_Progress" — этапы, время, ошибки, статусы (pending/in_progress/success/error), уведомления на каждом шаге.

## 3. КОМАНДЫ ДЛЯ МЕНЕДЖЕРОВ

**Основные команды:**

`/start` — Приветствие и справка по боту (персонализировано по роли)

`/help` — Полный список команд:
- 📊 Отчёты: `/bigreport`, `/reportmonth`, `/reportweek`, `/forecast`, `/report <ID>`
- 📨 Управление: `/broadcast`, `/tracker`, `/activate`, `/deactivate`, `/status`
- 🔧 Служебные: `/confirm`, `/cancel`, `/stop`

`/принять_[ID]` — Запуск онбординга студента (трекер + чат + Google Sheets)

`/пропустить_[ID]` — Передача студента следующему менеджеру

`/activate <ID>` — Активация существующего чата:
1. Проверка ID в KPI Ultra
2. Фильтрация участников (исключение ботов, менеджеров, @ultralina_zerocoder)
3. Выбор студента (автоматически если один, диалог если несколько)
4. Сохранение связи chat_id ↔ getcourse_id
5. Обновление "Випалина" (колонка G — invite), "Общий список new" (колонка G)

`/deactivate` — Очистка сегмента рассылок, отключение SLA-трекинга

`/status` — Проверка активации чата

`/broadcast [#тег] текст` — Рассылка во все чаты (или по сегменту):
1. Формирование списка целевых чатов
2. Превью с подтверждением
3. Таймаут 30 минут
4. После `/confirm` — отправка с задержкой 1 сек

`/confirm` — Подтверждение рассылки

`/cancel` — Отмена рассылки

`/stop` — Выход из любого диалога (отчёты, рассылка)

`/tracker <ID>` — Создание листов курса в трекере студента

## 4. ОТЧЁТНОСТЬ И АНАЛИТИКА

**`/bigreport`** — Комплексный отчёт:
- Руководитель: полная статистика, сравнение менеджеров, SLA отдела
- Менеджер: только свои студенты
- Структура: общая статистика, по менеджерам, прирост, активность чатов, ДЗ, требуют внимания (новые, без ДЗ, провисшие), SLA
- Исключения: "Не с нами", "ДЕЖУРНЫЙ"

**`/reportmonth`** — Месячный отчёт (текущий месяц):
- Выбор: свой/весь отдел/коллега
- Данные: статусы, динамика, конверсия (Новый→Учится, Учится→Завершил)

**`/reportweek`** — Недельный отчёт (7 дней):
- Активность, завершённые задачи, SLA, требуют внимания

**`/forecast`** — Прогноз доходимости:
- Расчёт на основе даты окупаемости, прогресса ДЗ, активности
- Группировка: доходные, риск, критично

**`/report <ID>`** — Детальный отчёт по студенту:
- Личные данные, обучение, прогресс, финансы (окупаемость, дедлайн), ссылки (трекер, чат, GetCourse)

## 5. SLA И ПРОИЗВОДИТЕЛЬНОСТЬ

**Отслеживание времени ответа:**
- Регистрация запроса студента + расчёт дедлайна (рабочее время: +60 мин, нерабочее: до следующего рабочего + 60 мин)
- Регистрация ответа менеджера + расчёт времени + проверка SLA (≤60 мин)
- Рабочее время: Пн-Пт 10:00-19:00 МСК
- Сохранение в "SLA_Records": дата, время, ID студента, менеджера, время ответа, SLA (ДА/НЕТ), invite

**Автоматические отчёты SLA:**
- Расписание: ежедневно 20:00 МСК
- Структура: общая статистика, по менеджерам, критические случаи
- Отправка: чат VIP-отдела + руководителю

**Ежедневные напоминания о студентах без контакта:**
- Расписание: Пн-Пт 11:00 МСК
- Логика: студенты без контакта ≥7 дней (исключение "ДЕЖУРНЫЙ", "Не с нами")
- Группировка по менеджерам + проверка ДЗ за неделю
- Формат: список студентов с днями без контакта, курсом, статусом ДЗ, ссылкой

**CSI (удовлетворённость):**
- Отправка опроса через 7 дней после создания чата
- Сбор оценки работы менеджера (⭐️1-5)

**Обновление дат обучения:**
- Синхронизация через формулы IMPORTRANGE
- Чтение дат из трекеров (начало, окупаемость, дедлайн) + обновление "Випалина" (колонки R, S, T)

## 6. УПРАВЛЕНИЕ ЧАТАМИ

**Создание чата:** CreateChatRequest(студент, менеджер, title="VIP | Имя") → invite-ссылка → приветствие → сохранение (chat_id, getcourse_id, invite, дата, участники)

**Мониторинг сообщений:** AI handler → проверка чата (chat_to_student) → определение отправителя → логирование ("Student_Messages") → обновление контакта ("Випалина") → SLA

**Типы сообщений:** text, photo, document, voice, video, sticker

**Последний контакт:** Автообновление колонки O ("Випалина") при сообщении, формат YYYY-MM-DD HH:MM:SS МСК. Используется для расчёта дней без контакта, напоминаний.

**Рассылки:** `/broadcast` → сегментация по полю "N" ("Випалина"), примеры: `#чатботы`, `#pro`, `#новички`, `#группа1`. Массовое обновление через Google Sheets.

## 7. УВЕДОМЛЕНИЯ

**О новом студенте:** Чат VIP-отдела (общее) + менеджер (полные данные + кнопки)

**Об этапах онбординга:** ⏳ Начало → ✅ Чат создан → 📨 Приветствие → 🎉 Завершено / ❌ Ошибка

**Об ошибках:** Приватность (инструкции + чат менеджер+бот), создание чата, трекера (описание + рекомендации)

**Системные:** Логирование в "System_Events" (startup, error, warning, info) + JSON данные

### 9.3. Обработка сбоев
**Стратегии:**

**1. Retry-логика:**
```python
for attempt in range(3):
    try:
        result = await operation()
        break
    except Exception as e:
        if attempt == 2:
            raise
        await asyncio.sleep(5)
```

**2. Graceful degradation:**
```python
try:
    await send_notification()
except Exception as e:
    logger.error(f"Уведомление не отправлено: {e}")
    # Продолжаем работу
```

**3. Логирование:**
```python
logger.error(f"Критическая ошибка: {e}", exc_info=True)
persistence.log_event('error', str(e), {'traceback': ...})
```
