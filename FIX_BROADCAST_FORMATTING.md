# Исправление форматирования в команде /broadcast

## 📋 Описание проблемы

При использовании команды `/broadcast` бот добавлял лишние символы (например, кавычки) к тексту сообщения. Например, команда:
```
/broadcast #тест текст
```
результат: `текст'` (с лишней кавычкой в конце)

## 🔍 Причина проблемы

Регулярное выражение `r'/broadcast\s+(.+)'` захватывало весь текст после команды, включая возможные кавычки, которые могли быть добавлены при копировании или вводе текста. Код не очищал текст от лишних символов.

## 🛠 Исправления

### В файле `vip_automation_main.py` (строки 1468-1474):

**Было:**
```python
content = match.group(1).strip()
```

**Стало:**
```python
content = match.group(1).strip()

# Убираем возможные кавычки в начале и конце
if content.startswith('"') and content.endswith('"') and len(content) >= 2:
    content = content[1:-1]
elif content.startswith("'") and content.endswith("'") and len(content) >= 2:
    content = content[1:-1]
```

## ✅ Результат исправления

1. **Корректная обработка текста** в команде `/broadcast`
2. **Удаление лишних кавычек** в начале и конце текста
3. **Сохранение оригинального форматирования** текста сообщения
4. **Код прошел проверку синтаксиса** без ошибок

## 🧪 Тестирование

После исправления:
- Команда `/broadcast #тест текст` отправляет сообщение `текст` без лишних символов
- Команда `/broadcast "Привет всем!"` отправляет сообщение `Привет всем!` без кавычек
- Команда `/broadcast 'Важное уведомление'` отправляет сообщение `Важное уведомление` без кавычек
- Форматирование текста сохраняется (переводы строк, пробелы и т.д.)
